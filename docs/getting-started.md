# Getting Started

## Create a Project

```bash
npm i -g @interactkit/cli
interactkit init my-app
cd my-app && pnpm install
```

This gives you:

```
my-app/
  interactkit/
    entities.xml            <- entity graph definition
    tools/                  <- tool handler files
  src/
    app.ts                  <- boot + configure
  package.json
  tsconfig.json
```

## Run

```bash
interactkit dev           # compile + run + watch for changes
```

This compiles `entities.xml` into typed TypeScript, builds, and starts your app. Changes to XML or handler files trigger automatic recompilation.

## Project Structure

### `interactkit/entities.xml`

The single source of truth for your entity graph. Defines entities, their state, tools, components, refs, and streams:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<graph xmlns="https://interactkit.dev/schema/v1" version="1" root="Agent">

  <entity name="Agent" type="base" description="Root agent">
    <components>
      <component name="brain" entity="Brain" />
      <component name="memory" entity="Memory" />
    </components>
    <tools>
      <tool name="ask" description="Ask a question" src="tools/ask.ts">
        <input><param name="question" type="string" /></input>
        <output type="string" />
      </tool>
    </tools>
  </entity>

  <entity name="Brain" type="llm" description="LLM brain">
    <describe>You are a helpful assistant.</describe>
    <executor provider="openai" model="gpt-4o-mini" />
    <refs>
      <ref name="memory" entity="Memory" />
    </refs>
  </entity>

  <entity name="Memory" type="base" description="Stores entries">
    <state>
      <fieldGroup name="entries" key="id">
        <field name="text" type="string" description="Content" />
      </fieldGroup>
    </state>
    <tools>
      <autotool name="store" on="entries" op="create" peerVisible="true" />
      <autotool name="getAll" on="entries" op="list" peerVisible="true" />
    </tools>
  </entity>

</graph>
```

### `interactkit/tools/`

Tool handler files referenced by the `src` attribute in XML. Each file exports a default async function:

```typescript
// interactkit/tools/ask.ts
import type { AgentEntity, AgentAskInput } from '../.generated/types.js';

export default async (entity: AgentEntity, input: AgentAskInput): Promise<string> => {
  return entity.components.brain.invoke({ message: input.question });
};
```

Handlers receive the entity instance (with typed state, components, refs, streams) and the validated input. The types are auto-generated from your XML.

### `src/app.ts`

Boot and configure the runtime:

```typescript
import { graph } from '../interactkit/.generated/graph.js';
import { PrismaDatabaseAdapter } from '@interactkit/prisma';
import { DevObserver } from '@interactkit/sdk';

const app = graph.configure({
  database: new PrismaDatabaseAdapter({ url: 'file:./interactkit.db' }),
  observers: [new DevObserver()],
});

await app.boot();

// Call tools on the typed proxy
const answer = await app.agent.ask({ question: 'What is TypeScript?' });
console.log(answer);
```

### `interactkit/.generated/`

Auto-generated by the compiler. Never edit these files:

| File | Contents |
|------|----------|
| `tree.ts` | Entity graph as nested objects |
| `registry.ts` | Zod schemas for state and tool inputs/outputs |
| `types.ts` | TypeScript interfaces for entities, proxies, inputs |
| `graph.ts` | Runtime instance with typed `configure()` |
| `handlers.ts` | Imports all `src`-defined tool handlers |

## CLI Commands

| Command | What it does |
|---------|-------------|
| `interactkit init <name>` | Create a new project |
| `interactkit compile` | Compile XML to typed TypeScript |
| `interactkit build` | Compile XML + TypeScript compilation |
| `interactkit dev` | Compile + run + watch for changes |
| `interactkit start` | Run the built application |

## Make Your First API Call

After `interactkit dev` is running, call a tool from your `app.ts`:

```typescript
const answer = await app.agent.ask({ question: 'What is the meaning of life?' });
console.log('Answer:', answer);
```

Or expose an HTTP API with `app.serve()`:

```typescript
await app.boot();
await app.serve({ http: 3000 });
// POST http://localhost:3000/agent/ask  { "question": "Hello" }
```

Every tool becomes an HTTP endpoint automatically. See [Deployment](deployment.md) for details.

## Entity Types

| Type | XML | Purpose |
|------|-----|---------|
| Base | `type="base"` | Standard entity with state and tools |
| LLM | `type="llm"` | Entity with an LLM executor, gets `invoke()` automatically |
| Long-Term Memory | `type="long-term-memory"` | Auto-generated memorize/recall/forget tools backed by a vector store |
| MCP | `type="mcp"` | Model Context Protocol server |

Long-term memory entities require no handler code. Add one to your graph and configure a `vectorStore` in `graph.configure()`:

```xml
<entity name="Memory" type="long-term-memory" description="Semantic memory" />
```

See [Entities](entities.md) for details.
